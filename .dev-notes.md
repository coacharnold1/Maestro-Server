# Maestro Development Notes

## Directory Structure
- **Git/Development**: `/home/fausto/Maestro-Server` - This is where we edit files
- **Running Instance**: `/home/fausto/maestro/web` - This is where the actual app runs from (NOT /home/fausto/maestro)

## Services
The Maestro system runs as two systemd services:
- `maestro-web.service` - Main web UI (port 5003)
- `maestro-admin.service` - Admin interface (port 5004)

## Workflow for Changes

1. **Edit files** in `/home/fausto/Maestro-Server/`
2. **Copy to running instance**: 
   ```bash
   sudo cp /home/fausto/Maestro-Server/templates/index.html /home/fausto/maestro/web/templates/index.html
   ```
3. **Restart the web service**:
   ```bash
   sudo systemctl restart maestro-web.service
   ```
4. **Check status** (if needed):
   ```bash
   sudo systemctl status maestro-web.service
   ```

## Useful Commands
- View logs: `sudo journalctl -u maestro-web.service -f`
- Stop service: `sudo systemctl stop maestro-web.service`
- Start service: `sudo systemctl start maestro-web.service`
- Restart both: `sudo systemctl restart maestro-web.service maestro-admin.service`
